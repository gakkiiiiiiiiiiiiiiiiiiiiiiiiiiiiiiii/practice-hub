# 问题排查指南

## 当前错误

### 1. Trace is not defined
这是微信开发者工具的基础库已知问题，不影响功能。建议：
- 升级微信开发者工具到最新版本
- 在项目设置中选择基础库版本 >= 3.0.2

### 2. Cannot set property 'SDKVersion' of undefined
这也是基础库相关的问题，通常不影响页面渲染。

## 页面空白问题排查

### 已修复的问题

1. ✅ 移除了所有组件依赖（AppIcon, AppCountdown）
2. ✅ 移除了接口调用（get 函数）
3. ✅ 移除了 store 依赖（useBankStore）
4. ✅ 使用内联样式替代 SCSS 变量
5. ✅ 数据在初始化时就有值
6. ✅ 简化了导航栏高度计算

### 检查步骤

1. **检查控制台**
   - 打开微信开发者工具的控制台
   - 查看是否有红色错误
   - 查看是否有警告信息

2. **检查页面结构**
   - 在开发者工具中查看 Elements/AppData
   - 检查 DOM 是否已渲染
   - 检查数据是否正确

3. **检查样式**
   - 查看 Computed 样式
   - 检查是否有 display: none 或 visibility: hidden
   - 检查 z-index 是否正确

4. **测试简化页面**
   - 临时将 `pages.json` 中的首页改为 `pages/index/index-debug`
   - 如果调试页面能显示，说明是组件问题
   - 如果调试页面也空白，说明是基础配置问题

### 可能的解决方案

1. **清除缓存**
   ```bash
   # 删除 node_modules 和重新安装
   rm -rf node_modules
   npm install
   ```

2. **重新编译**
   - 在微信开发者工具中点击"编译"
   - 或者使用命令行重新构建

3. **检查 manifest.json**
   - 确保配置正确
   - 检查是否有语法错误

4. **检查 pages.json**
   - 确保页面路径正确
   - 检查是否有配置错误
